Troubleshooting

‚ùå Docker daemon not running
Solution: Start Docker Desktop or run sudo systemctl start docker (Linux)
‚ùå Port 8080 already in use
Solution: Stop the conflicting service or change OpenChoreo port with --set service.port=8081
‚ùå Pods stuck in Pending state
Solution: Check resources: kubectl describe pod POD_NAME -n openchoreo-system. May need to increase cluster resources.
‚ùå Helm installation fails
Solution: Verify Helm version (need 3.x+), check cluster access: kubectl get nodes
‚ùå Cannot access http://localhost:8080
Solution: Check ingress: kubectl get ingress -n openchoreo-system. May need to run kubectl port-forward svc/openchoreo-ui 8080:80 -n openchoreo-system
