Air-gapped Installation

Deploy OpenChoreo in completely disconnected environments without internet access. Designed for high-security environments and classified networks.

Includes offline image bundles, private registry setup, and complete offline documentation for secure environments.

‚è±Ô∏è Time: 2 hours
üìã Prerequisites: Private registry
üíæ Resources: 16GB RAM, offline storage
üéØ Best for: Secure, classified envs
Phase 1: Image Preparation
‚ñº
# On connected machine
wget https://openchoreo.io/releases/v1.0.0/openchoreo-images-v1.0.0.tar.gz
wget https://openchoreo.io/releases/v1.0.0/openchoreo-charts-v1.0.0.tar.gz

scp openchoreo-*.tar.gz airgap-host:~/
# Files transferred to air-gapped environment

# On air-gapped machine
tar -xzf openchoreo-images-v1.0.0.tar.gz
./load-images.sh --registry private-registry.company.com
# Output: ‚úÖ Loaded openchoreo/api:v1.0.0 ‚úÖ Loaded openchoreo/ui:v1.0.0 ‚úÖ All 15 images loaded to private-registry.company.com
Phase 2: Offline Installation
‚ñº

cd openchoreo-charts

helm install openchoreo ./platform \ --namespace openchoreo-system \ --create-namespace \ --values values-airgapped.yaml \ --set global.imageRegistry=private-registry.company.com \ --wait
# Output: NAME: openchoreo STATUS: deployed OpenChoreo running in air-gapped mode
Phase 3: Install Sample Add-on
‚ñº
helm install openchoreo-monitoring ./monitoring \ --namespace openchoreo-system \ --set global.imageRegistry=private-registry.company.com \ --values values-airgapped-monitoring.yaml
# Output: Monitoring addon installed (air-gapped mode)

kubectl logs -n openchoreo-system deployment/prometheus-server | grep -i "connection"
# Output: (no external connection attempts)
Phase 4: Cleanup
‚ñº
helm uninstall openchoreo-monitoring -n openchoreo-system
helm uninstall openchoreo -n openchoreo-system
# Output: Platform removed

kubectl delete namespace openchoreo-system
# Output: namespace "openchoreo-system" deleted
